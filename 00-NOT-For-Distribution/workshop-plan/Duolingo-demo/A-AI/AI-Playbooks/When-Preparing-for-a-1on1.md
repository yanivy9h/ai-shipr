# Playbook: When Preparing for a 1:1

**Paste this into Claude Code the morning before a 1:1 with your manager, skip-level, or key stakeholder.**
Use it instead of going in with a blank agenda or a status dump.

This playbook builds a decision-first agenda from your active initiatives, pending decisions, and blockers.

---

## Instructions to Claude

Read the following files before responding:
- `R-Relationships/Me/PM-Profile.md`
- `R-Relationships/Stakeholders/Stakeholders.md`
- `Learning.md`
- `AI/AI-Skills/1on1-Prep.md`
- All files in `I-Initiatives/` (scan for pending decisions, blockers, At Risk status)
- All files in `H-Hypotheses/` (scan for anything that needs a decision)

Then:

**Step 1 — Scan for Decisions**
Find all open decisions across active initiatives.
Prioritize: what is blocking progress right now.
Surface the top 2-3 items that require this person's input or authority.

**Step 2 — Run 1on1-Prep**
Using my context below, build the full 1:1 agenda.
Structure: Decisions Required → Escalations → Initiative Updates → Their Agenda.
Allocate time to each section within the total meeting length.

**Step 3 — Escalation Frame**
If I've flagged anything as an escalation, produce the full escalation frame:
Situation → Impact → What I've tried → My recommendation → Specific ask.

**Step 4 — Write**
Ask: "Do you want me to save this agenda?"
If yes: write to `R-Relationships/Stakeholders/Meeting-Log.md` as a new entry.
After the 1:1, remind me to log decisions made back into the relevant initiative files.
