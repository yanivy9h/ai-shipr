# Pre-Workshop Checklist

Complete these steps before the session. The whole setup takes about 20 minutes.

---

## Step 1 — Install Obsidian

1. Go to obsidian.md
2. Download and install the free version for your operating system (Mac / Windows)
3. Open it — you will see the welcome screen asking you to create or open a vault

You do not need to create a vault yet.

---

## Step 2 — Install Claude Code

Claude Code is the AI interface you will use during the workshop.

1. Go to claude.ai and create a free account if you do not have one
2. Follow the Claude Code installation instructions at claude.ai/code
3. You will need to install it as a CLI tool — it runs in your terminal / command line
4. Once installed, open a terminal and type `claude` — you should see the Claude Code prompt

**Note:** Claude Code requires a paid Anthropic plan for full use. A free account gives limited access. If you want full functionality during and after the workshop, an Anthropic subscription (starting at ~$20/month) is needed. You can try it on the free tier during the session.

---

## Step 3 — Download and Open the AI-SHIPR Starter Package

1. Download the ZIP file from the link in your welcome email
2. Unzip it — you will get a folder called `AI-SHIPR`
3. Open Obsidian
4. Click "Open folder as vault"
5. Select the `AI-SHIPR` folder
6. The vault opens — you should see the folder structure on the left sidebar

If the sidebar shows folders like `S-Strategy`, `I-Initiatives`, `H-Hypotheses` — you are set up correctly.

---

## Step 4 — Connect Claude Code to Your Vault

1. Open your terminal
2. Navigate to your `AI-SHIPR` folder:
   `cd /path/to/AI-SHIPR`
3. Type `claude` and press Enter
4. Claude Code should start, reading from the `CLAUDE.md` file in the folder
5. Type `/today` and press Enter — Claude Code will attempt to run the daily briefing (it will flag missing data, which is expected)

If this works, you are ready for the session.

---

## Verify You Are Ready

- [ ] Obsidian installed and vault opens correctly
- [ ] Claude Code installed and responds to commands
- [ ] `/today` runs without an error (a "missing data" response is fine)
- [ ] You have a product in mind to work with during the session

---

## Having Trouble?

Reply to the welcome email with what step you are on and what the error says. Yaniv will help you before the session.

Do not arrive to the workshop with a broken setup — it costs everyone time and you will miss half the value.
