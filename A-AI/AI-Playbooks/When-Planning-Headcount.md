# Playbook: When Planning Headcount

**Paste this into Claude Code when you need to make a headcount decision — a new hire, backfill, or team restructure.**
Use it before writing a job brief or making a request to leadership.

This playbook grounds the headcount ask in strategy — not in workload alone.

---

## Instructions to Claude

Read the following files before responding:
- `S-Strategy/Strategic-Bets.md`
- `R-Relationships/Team/Roster.md`
- All files in `R-Relationships/Team/PM-Profiles/`
- All files in `I-Initiatives/`
- `A-AI/AI-Skills/Resource-Allocation-Mapper.md`
- `A-AI/AI-Skills/Hiring-Brief-Builder.md`

Then:

**Step 1 — Run Resource-Allocation-Mapper**
Produce the current allocation map:
- What each PM owns vs. what the bets require
- Where the gap is — specific bet, not just "we're busy"
- What happens if the gap is not filled (stated as a strategic consequence, not a capacity complaint)

**Step 2 — Build the Case**
Structure the headcount ask for leadership:
- Strategic gap: [which bet is under-covered and why it matters]
- Current workaround and its cost: [what are we doing instead and what is it costing us]
- What this hire enables: [specific — initiative unlocked, bet covered, PM freed up]
- Risk of not hiring: [concrete — not "we'll be stretched"]

**Step 3 — Run Hiring-Brief-Builder**
Using the context below, build the job brief for this role:
- Role level and scope
- Success in 90 days
- Must-haves
- Recruiter brief

**Step 4 — Write**
Ask: "Do you want me to save this hiring brief?"
If yes: write to `R-Relationships/Team/Hiring/[Role-Title]-brief.md`
