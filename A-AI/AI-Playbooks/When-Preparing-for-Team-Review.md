# Playbook: When Preparing for Team Review

**Paste this into Claude Code at the start of the week — before your first team sync, standup, or 1:1.**
Use it to get a clear picture of team state before any conversations happen.

This playbook replaces ad-hoc "how are things going?" check-ins with a structured read from the system.

---

## Instructions to Claude

Read the following files before responding:
- `R-Relationships/Team/Roster.md`
- All files in `R-Relationships/Team/PM-Profiles/`
- All files in `R-Relationships/Team/1on1-Log/` (most recent entry per PM)
- All files in `I-Initiatives/` (scan for status, blockers, At Risk)
- `S-Strategy/Strategic-Bets.md`
- `A-AI/AI-Workflows/Team-Review.md`

Then:

**Step 1 — Run Team-Review**
Generate the full team review briefing:
- Team snapshot (load, blockers, decisions needed per PM)
- Blockers requiring my action
- Bet coverage check
- 1:1 queue for this week

**Step 2 — Run Team-Health-Check**
Using `A-AI/AI-Skills/Team-Health-Check.md`:
Surface any morale signals, overload risks, or systemic patterns from the 1:1 logs.

**Step 3 — Prioritize My Actions**
Based on the review, give me a ranked list of what I need to do this week as a team lead:
- Blockers to clear (for whom, by when)
- 1:1s to run (with suggested focus per PM)
- Decisions to make (with context)
- One thing to watch but not act on yet

**Step 4 — Write**
Ask: "Do you want me to save this week's team review?"
If yes: write to `P-Proof/TeamReview-[date].md`
