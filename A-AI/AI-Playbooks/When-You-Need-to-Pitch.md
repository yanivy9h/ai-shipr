# Playbook: When You Need to Pitch

**Paste this into Claude Code when you need to secure investment, headcount, or prioritization for an initiative.**
Use it before any exec presentation, leadership review, or funding conversation.

This playbook builds the business case first, then converts it into a leadership presentation narrative.

---

## Instructions to Claude

Read the following files before responding:
- `S-Strategy/Strategic-Bets.md`
- `S-Strategy/KPIs.md`
- `S-Strategy/Constraints.md`
- `R-Relationships/Stakeholders/Stakeholders.md`
- `AI/AI-Skills/Business-Case-Builder.md`
- `AI/AI-Skills/Pitch-Deck-Builder.md`
- The initiative file I name below (from `I-Initiatives/`) if it exists

Then:

**Step 1 — Run Business-Case-Builder**
Using my input below, build the full business case.
Enforce: quantified impact, cost estimate with range, strategic bet link, and the hardest objection pre-addressed.
If any required input is missing, ask me for it — do not fill in placeholders.

**Step 2 — Validate**
Before moving to the deck, confirm:
"Here is the business case. Is the impact quantification and the ask accurate before I build the deck?"
Wait for my confirmation.

**Step 3 — Run Pitch-Deck-Builder**
Convert the confirmed business case into a 9-slide narrative brief.
Produce: slide-by-slide brief with headline, content, and "what they should ask next" for each slide.
Include delivery notes.

**Step 4 — Write**
Ask: "Do you want me to save the business case and deck brief?"
If yes:
- Write business case to `P-Proof/BusinessCase-[initiative-name].md`
- Write deck brief to `P-Proof/PitchDeck-[initiative-name].md`
