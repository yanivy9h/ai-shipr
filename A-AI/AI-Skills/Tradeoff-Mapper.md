# Skill: Tradeoff Mapper

## Purpose
Map tradeoffs between two or more options clearly and comparatively.
Force the real costs of each choice into the open before a decision is made.

## When to Use
- When options exist but the decision is stalling
- Before using Decision Architect — use this to prepare the tradeoff input
- When a team is debating without a shared frame
- When an option looks dominant but hasn't been stress-tested

## Behavior Rules
- Require at least 2 options — this skill has no purpose with a single option
- Every criterion must appear for every option — no skipping
- Do not favor any option — the goal is clarity, not advocacy
- Distinguish short-term vs long-term implications explicitly
- Recommendation must be conditional, not absolute — state the condition
- Flag if options are not actually distinct (false tradeoff)

## Input Expectations
Provide:
- 2–4 options to compare
- Context: what decision this supports

Optional:
- Criteria you want evaluated
- Stakeholder constraints
- Time horizon for the decision

## Output Structure

### Options Summary

| A      | ...           | ...            |
| B      | ...           | ...            |

### Comparative Criteria
| Criterion        | Option A | Option B | Notes |
|-----------------|----------|----------|-------|
| Speed to value  | ...      | ...      | ...   |
| Risk level      | ...      | ...      | ...   |
| Resource cost   | ...      | ...      | ...   |
| Reversibility   | ...      | ...      | ...   |
| Strategic fit   | ...      | ...      | ...   |

### What Each Option Gives Up
- Option A sacrifices: [...]
- Option B sacrifices: [...]

### Time Horizon Implications
- Short-term (0–3 months): [A vs B]
- Long-term (6–12 months): [A vs B]

### Conditional Recommendation
> Choose Option A if [specific condition].
> Choose Option B if [specific condition].
> Do not decide yet if [blocking uncertainty that must be resolved first].
