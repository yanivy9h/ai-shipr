# Templates

Reusable document structures for outputs you generate repeatedly.

Drop a template here when you find yourself writing the same type of document more than twice.
AI-SHIPR skills reference these to produce consistent, on-brand output every time.

---

## Current Templates (6)

| Template | Purpose |
|---|---|
| `template-1on1.md` | 1:1 meeting prep: pre-meeting read + structured decision-first agenda |
| `template-decision-log.md` | Decision documentation: statement, criteria, tradeoffs, and outcome |
| `template-okr.md` | OKR framing: objective + key results with metric specificity and bet linkage |
| `template-retro.md` | Retrospective structure: preserve / improve / actions |
| `template-sprint-review.md` | Sprint review: committed scope vs. shipped scope with outcome signals |
| `template-stakeholder-update.md` | Stakeholder communication: one thing they need to know framework |

---

## What belongs here

- PRD / Feature Spec templates
- Stakeholder update formats
- Decision log entry structure
- Sprint review / retrospective formats
- OKR and roadmap framing templates
- 1:1 agenda templates
- Launch checklist templates
- Any document type you produce regularly

---

## How to use

Name files clearly: `template-prd.md`, `template-stakeholder-update.md`, etc.
Reference them in skills with: `Read the template at Information/Templates/template-prd.md`
