# R-Relationships

The people layer. Everything the system needs to know about the PM, their users, their stakeholders, and the history of key conversations.

Agents read from here to make outputs feel personal — not generic. Without this folder, every response is one-size-fits-all. With it, the system knows who it's working for and who it needs to persuade.

---

## Role in the system

R is the human context layer. It does not track work (that's I-Initiatives). It tracks people — their motivations, communication styles, and what they need to say yes.

---

## Folder structure

Three subfolders, each grouping a different category of people:

### `Me/` — The PM
| File | What it is | Who fills it |
|---|---|---|
| `Me/PM-Profile.md` | Role, responsibilities, decision ownership, operating preferences, and where you want leverage from AI | PM fills this during setup |
| `Me/PM-Voice.md` | Communication style extracted from real writing samples — makes AI outputs sound like you | Generated by `Create-PM-Voice` skill |

### `Users/` — External users and customers
| File | What it is | Who fills it |
|---|---|---|
| `Users/Personas.md` | Structured user personas: role, job-to-be-done, pain points, emotional drivers, success definition | PM fills or `Persona-Builder` skill generates |

### `Stakeholders/` — Internal stakeholders
| File | What it is | Who fills it |
|---|---|---|
| `Stakeholders/Stakeholders.md` | Stakeholder profiles: what they optimize for, biggest fears, objections, communication style, what they need to say yes | PM fills or `Stakeholder-Profiler` skill generates |
| `Stakeholders/Meeting-Log.md` | Structured record of key meetings: participants, initiatives discussed, decisions made, objections raised, action items | PM fills after meetings |

---

## Who reads this folder

| Tool | What it uses |
|---|---|
| `Stakeholder-Translator` agent | Reads `Stakeholders/Stakeholders.md` to reframe initiatives in stakeholder language |
| `Narrative-Refiner` skill | Reads `Me/PM-Voice.md` to match tone and communication style |
| `1on1-Prep` skill | Reads `Stakeholders/Stakeholders.md` and `Stakeholders/Meeting-Log.md` to build the agenda |
| `Sprint-Summarizer` skill | Reads `Me/PM-Voice.md` to produce summaries that sound like you |
| `Cross-Team-Mapper` skill | Reads `Stakeholders/Stakeholders.md` to map owners and relationships |
| `Persona-Builder` skill | Writes output back into `Users/Personas.md` |
| `Create-PM-Voice` skill | Writes output into `Me/PM-Voice.md` |

---

## Setup priority

Fill in this order on day one:
1. `Me/PM-Profile.md` — defines the operating context for everything
2. `Stakeholders/Stakeholders.md` — needed before any alignment or communication work
3. `Users/Personas.md` — needed before any discovery or hypothesis work
4. `Me/PM-Voice.md` — run `Create-PM-Voice` skill once you have 3–5 writing samples

`Stakeholders/Meeting-Log.md` fills naturally over time.
