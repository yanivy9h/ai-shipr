# Shared Templates

Org-standard document formats that all PMs use.
When a PM generates a PRD, stakeholder update, or OKR — they use these templates, not personal versions.
This ensures consistency across the portfolio and makes cross-PM work readable.

---

## What belongs here

- **PRD template** — the org-standard format for feature specs
- **Stakeholder update template** — the standard format for shipping news to stakeholders
- **OKR framing template** — how the org writes objectives and key results
- **1:1 agenda template** — standard 1:1 structure (if different from the default AI-SHIPR template)
- **Decision log format** — how decisions are documented across the org
- **Sprint review format** — org-standard sprint review structure
- **Any document format used across more than one product**

---

## How to use

Drop `.md` template files here.
Reference them in skills with: `Read the org template at ../vp/shared/I-Information/Templates/[filename].md`

Personal or product-specific templates stay in `pm-[name]/I-Information/Templates/`.
